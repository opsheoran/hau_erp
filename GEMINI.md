# Core Approval Flows

This project features a strict hierarchical approval flow for various modules. **When implementing any page or logic within these modules, you MUST adhere strictly to the following progression and database flags:**

## 1. Admission & Academics Module
These tables govern student lifecycles, committees, and course management:

*   **Advisory Committee Management (SMS_Advisory_Committee_Mst)**
    *   **Flow:** HOD -> College Dean -> Dean PGS
    *   **DB Flags:** hod_approval, college_deanapproval, pprovalstatus (used for Dean PGS).
*   **Course Addition / Withdrawal (SMS_CourseAdditionWithdrawal_Mst)**
    *   **Flow:** Teacher/Student (Submission) -> College Dean
    *   **DB Flags:** Approval (Initial flag), dean_approv (Final Dean approval).
*   **Student Registration / No-Dues Clearance (SMS_StuCourseAllocation_Approval_staffwise)**
    *   **Flow:** This module features a massive, parallel/sequential cross-departmental clearance flow.
    *   **DB Flags:** Teacher_AprrovalStatus, Adv_AprrovalStatus (Advisor), Lib_AprrovalStatus (Library), DSW_AprrovalStatus (Directorate of Students Welfare), ee_AprrovalStatus (Accounts), Dean_AprrovalStatus (College Dean), Deanpgs_AprrovalStatus (Dean PGS), and finally Registrar_AprrovalStatus.

## 2. Examination & Results Module
These tables govern the entry, verification, and finalization of student marks and grades:

*   **Regular & Revised Marks Entry (SMS_StuExamMarks_Dtl, sms_stuexammarks_dtl_revised)**
    *   **Flow (UG):** Teacher (Submission) -> HOD -> COE -> Registrar
    *   **Flow (PG/PhD):** Teacher -> HOD -> Dean PGS -> COE -> Registrar
    *   **DB Flags:** islockedbyHOD (Marks locked by HOD), IsApprovedByHOD, IsApprovedByDeanPGS, IsApprovedByCOE, IsApprovedByRegistrar.
*   **Re-Evaluation Marks Entry (sms_stuexammarks_dtl_reval)**
    *   **Flow:** Teacher -> HOD -> COE -> Registrar
    *   **DB Flags:** IsApprovedByHOD, IsApprovedByCOE, IsApprovedByRegistrar (Dean PGS is explicitly skipped).
*   **Final Exam Grading (SMS_ExamGrading)**
    *   **Flow:** HOD -> Registrar
    *   **DB Flags:** IsHODApproval, IsRegistrarApproval.
*   **I-Grade (Incomplete Grade) Assignments (SMS_Igrade_Mst)**
    *   **Flow:** Teacher -> HOD -> College Dean
    *   **DB Flags:** Hod_approv, dean_approv.
*   **Supplementary Examinations (SMS_Suppliment_Mst)**
    *   **Flow:** Teacher/Submission -> College Dean
    *   **DB Flags:** dean_approv.

**UI Standard Implementation:** When building these approval pages, always mimic the exact form controls (text boxes, dropdowns, buttons, grid structure) of the provided live templates, while utilizing the project's existing CSS classes (e.g. .form-table, .textbox, .w-200, .button-common, .table-grid-view).
